PWM

Overview

PWM is the interface for pulse-width modulation for all MCUs. Drivers like LED RGB can use this interface to be portable between different MCUs.

Initialization

Initialization is not done using this interface. Instead, the specific PWM driver for a given MCU should be used for configuration. However, there are the setPeriod member functions that might only be called during initialization in some applications.

Example Derived Class

This example showcases a simplified representation of a memory-mapped timer peripheral in a microcontroller. The PWM interface presented here offers an illustration of its usage.

In the code, a pseudo-memory-mapped timer peripheral is simulated through a structure named MCU_TIMER_T, which contains an auto-reload and match register for timer output channel 0. Of course, a real peripheral will have many other registers, or may work differently than this common scenario. For the purpose of this example, an actual instance of this simulated peripheral, MCU_TIMER0, is created. Typically, in real microcontrollers, the address for such a peripheral would be obtained from the datasheet and hardcoded into the #define.

The PWM class, derived from the EDF::PWM base class, provides an abstraction for interacting with a PWM output.

Example
/* Simulate a Timer memory mapped peripheral, here is just a couple "registers" */
struct MCU_TIMER_T {
    volatile uint32_t AUTO_RELOAD;
    volatile uint32_t MATCH_CH0;
};

/* For this example, we need a real instance of the fake memory mapped peripheral */
static MCU_TIMER_T MCU_TIMER0;

/* Normally the definition is a hardcoded address that comes from the datasheet */
#define TIMER0 (&MCU_TIMER0)

class PWM final : public EDF::PWM {
private:
    MCU_TIMER_T* timer;
    uint32_t channel;
public:
    PWM( MCU_TIMER_T* timer, uint32_t channel ) : timer(timer), channel(channel) {}
    virtual ~PWM() = default;

    virtual void setPeriod_MHz( uint32_t MHz )                { setPeriod_kHz( MHz * 1000 ); }
    virtual uint32_t getPeriod_MHz()                    const { return getPeriod_kHz() / 1000; }

    virtual void setPeriod_kHz( uint32_t kHz )                { setPeriod_Hz( kHz * 1000 ); }
    virtual uint32_t getPeriod_kHz()                    const { return getPeriod_Hz() / 1000; }

    virtual void setPeriod_Hz( uint32_t Hz )                  { timer->AUTO_RELOAD = Hz; /* actually do math */  }
    virtual uint32_t getPeriod_Hz()                     const { return /* sysclk */48000000; }

    virtual void setPeriod_ticks( uint32_t ticks )            { timer->AUTO_RELOAD = ticks; }
    virtual uint32_t getPeriod_ticks()                  const { return timer->AUTO_RELOAD; }

    virtual void setDutyCyclePercent( uint8_t percent )       { setDutyCycleTicks( (getPeriod_ticks() * percent) / 100 ); }
    virtual uint8_t getDutyCyclePercent()               const { return static_cast<uint8_t>((getDutyCycleTicks() * 100) / getPeriod_ticks()); }

    virtual void setDutyCycleTicks( uint32_t ticks )          { if( channel == 0 ) timer->MATCH_CH0 = ticks; /* etc..*/ }
    virtual uint32_t getDutyCycleTicks()                const { return (channel == 0) ? timer->MATCH_CH0 : 0; /* etc..*/ }
};

setPeriod_MHz()

Set the period in terms of Megahertz. The implementation for a specific microcontroller should consider using Assert to ensure the provided value can be achieved.

This typically will affect more than a single timer output channel. So be aware when configuring multiple PWM outputs which hardware timers should be used if outputs do not share the same period.
Example
PWM pwm( TIMER0, 0 );
pwm.setPeriod_MHz( 5 );

getPeriod_MHz()

Get the current period value set in terms of Megahertz.

This value might be 0 if the period is less than 1 MHz
Example
PWM pwm( TIMER0, 0 );
uint32_t MHz = pwm.getPeriod_MHz();

setDutyCyclePercent()

Set the duty cycle in terms of a percentage.

Expected values are 0-100
Example
PWM pwm( TIMER0, 0 );
pwm.setDutyCyclePercent( 50 );

getDutyCyclePercent()

Get the duty cycle in terms of a percentage.

Example
PWM pwm( TIMER0, 0 );
uint8_t dutyCyclePercent = pwm.getDutyCyclePercent();

setDutyCycleTicks()

Set the duty cycle in terms of timer ticks.

Can use this in combination with getPeriod_ticks() for a percentage with more resolution than with setDutyCyclePercent(). EX: 50.5% duty cycle could be set using setDutyCycleTicks( (505 * 1000) / getPeriod_ticks() )
Example
PWM pwm( TIMER0, 0 );
pwm.setDutyCycleTicks( 960 / 2 );

getDutyCycleTicks()

Get the current duty cycle in terms of timer ticks.

Can use this in combination with getPeriod_ticks() for a percentage with more resolution than with getDutyCyclePercent(). EX: if at 50.5 percent uint32_t percentTimes10 = (getDutyCycleTicks() * 1000) / getPeriod_ticks()
Example
PWM pwm( TIMER0, 0 );
uint32_t dutyCycleTicks = pwm.getDutyCycleTicks();
