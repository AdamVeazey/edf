Stack<T, N>

Template arguments

T = (T)ype
N = Maximum (N)umber of elements the stack can hold

T Needs to be copyable, and helps if T is also default constructable.

Overview

This is a container adapter class to provide a LIFO data structure. Stack uses a Vector as the underlying container. Stack has a maximum number of elements (N) the stack can "grow" to.

There are 3 groups of member functions:

  1. Is Questions

  2. Capacity

  3. Operations

Initialization

Default initialization requires type T to be default constructable. Otherwise list-initialization is required.

Example: T is default constructable
EDF::Stack<int, 4> stack;
Example: list-initialization with a custom type
struct CustomType{
    int value;
    CustomType( int v ) : value(v) {}
};

EDF::Stack<CustomType, 3> stack2 = {
    CustomType( 0 ),
    CustomType( 0 ),
    CustomType( 0 )
};
stack2.clear();
If using list-initialization on a type T without a default constructor. First fill up the entire stack using list-initialization, then call clear()

Is Questions

These member functions provide yes/no answers to the current state of the stack.

isEmpty()

Equivalent to length() == 0
Provided to make a common conditional check slightly more readable, as well as consistency with other EDF containers.

Example
EDF::Stack<int, 4> stack;
while( !stack.isEmpty() ){
    /* ... */

isFull()

Since a stack doesn’t grow the underlying container, you should check if it is full before adding an element.

Example
EDF::Stack<int, 4> stack;
if( !stack.isFull() ) {
    /* ... */
}

Capacity

Provides information about how many current elements are in the stack, as well as the max number of elements the stack can hold.

length()

Returns the number of elements currently in the stack.

Example
EDF::Stack<int, 4> stack;
auto& nElementsInStack = stack.length();

maxLength()

Returns template parameter N. It acts the same as a compile time constant. No need to remember how a macro for the stack max number of elements is named.

Example
EDF::Stack<int, 4> stack;
constexpr auto maxNumberOfElements = stack.maxLength();

Operations

Many, if not all, of the Operation member functions use EDF_ASSERTD to ensure the operation will be successful. Remember that the assert condition is only checked in Debug mode.

peek()

Returns the last value inserted into the stack without modifying the number of elements in the stack.

Example
EDF::Stack<int, 4> stack;
stack.push( 1 );
auto& lastElement = stack.peek();
Name is peek instead of "top" for consistency with Queue. It gives a better picture of what it does. Peek allows you to peek, look, read, the element you are likely to remove next using pop() without actually popping the value.

push( value )

Add element to the end of the stack.

Example
EDF::Stack<int, 4> stack;
stack.push( 1 );

emplace( args…​ )

Add element to the end of the stack by constructing the element in place using placement new.

Example
EDF::Stack<int, 4> stack;
stack.emplace( 2 );

pop()

Remove element from the end of the stack then return a copy of the value.

Example
EDF::Stack<int, 4> stack;
    auto mostRecentElement = stack.pop();

clear()

Calls destructor for each element in stack. Sets the number of elements to 0.

Example
EDF::Stack<int, 4> stack;
stack.clear();
